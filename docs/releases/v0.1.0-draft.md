# AutoForge v0.1.0 (Draft) / AutoForge v0.1.0（草稿）

First public release of AutoForge.  
AutoForge 首个公开版本。

## Highlights / 版本亮點

- Added GitHub Actions CI workflow (`.github/workflows/ci.yml`)  
  新增 GitHub Actions CI 工作流（`.github/workflows/ci.yml`）
  - Python matrix: 3.10 / 3.11 / 3.12  
    Python 版本矩阵：3.10 / 3.11 / 3.12
  - Lint checks: ruff / black / isort  
    代码检查：ruff / black / isort
  - Test execution: pytest  
    测试执行：pytest

- Added PyPI publish workflow (`.github/workflows/publish-pypi.yml`)  
  新增 PyPI 发布工作流（`.github/workflows/publish-pypi.yml`）
  - Build artifacts with `python -m build`  
    使用 `python -m build` 构建发布文件
  - Publish via `pypa/gh-action-pypi-publish`  
    通过 `pypa/gh-action-pypi-publish` 自动发布
  - Supports trusted publishing with OIDC (`id-token: write`)  
    支持 OIDC Trusted Publishing（`id-token: write`）

- Improved documentation  
  文档内容增强
  - Enhanced `README.md` with CI/PyPI badges and screenshots section  
    `README.md` 新增 CI/PyPI 徽章与截图区块
  - Added CI/release guardrails and release checklist  
    新增 CI/发布保护策略与发版检查清单
  - Synced `README_zh.md` with corresponding updates  
    `README_zh.md` 已同步相同更新

- Added launch blog draft  
  新增发布文章草稿
  - `docs/blog/autoforge-launch-draft.md`

## Notes / 说明

- Package and CLI entry point remain `mltune` for compatibility.  
  为了兼容性，包名与 CLI 入口保持为 `mltune`。
- Branding and user-facing docs are updated to AutoForge.  
  品牌与用户可见文档已更新为 AutoForge。

## Upgrade / Install / 升级与安装

```bash
pip install -U autoforge-mltune
```
